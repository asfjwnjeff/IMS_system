## 1. 产品概述

### 1.1 产品定位

IMS（Insurance Management System）保险管理系统，面向供应链物流场景，管理货物运输险的投保、保单、理赔全流程。主要用户为货运代理企业的业务人员和保险专员。

### 1.2 核心用户角色

| 角色 | 职责 |
|------|------|
| **业务人员** | 创建投保单、填写货物/运输信息、选择保险产品、发起审批、查看保单 |
| **保险专员** | 后台配置保险产品、汇率、费率；核保回填保单信息 |
| **审批人员** | 审核投保申请，通过/拒绝 |
| **管理员** | 系统配置、用户/角色/字典管理 |

### 1.3 技术栈

| 层面 | 技术 |
|------|------|
| 框架 | Next.js 16 (App Router + Turbopack) |
| 语言 | TypeScript |
| UI | shadcn/ui + Tailwind CSS |
| 状态管理 | React Context + useReducer |
| 数据库 | SQLite (sql.js WASM) + Drizzle ORM |
| 图标 | Lucide React |
| 提示 | Sonner (toast) |
| 端口 | **5002** |

## 4. 基础信息（★ 本次核心优化）

### 4.1 保费汇率配置

**旧设计**：汇率按"保险公司 + 币种"维度管理（一条记录同时绑定保险公司和币种）。  
**新设计**：汇率仅按**币种**维度统一管理，不再区分保险公司。

**字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 币种 | Select | 中文币种名（美元/欧元/日元/港币/…），必填 |
| 汇率 | Number | 对人民币汇率，6 位小数精度，必填 |
| 生效日期 | Date | 必填 |
| 失效日期 | Date | 必填 |
| 创建人 | Text | — |
| 创建时间 | Text | 自动生成 |

**业务规则**：
- 人民币汇率恒为 1，不存入汇率表
- 保费试算时自动按投保单币种匹配对应汇率
- "保险公司"字段已废弃，旧数据迁移为空字符串

### 4.2 保费费率配置

**旧设计**：费率按"保险公司 + 货物类型"维度管理。  
**新设计**：费率按**保险产品**维度管理，保险专员配置产品库，业务人员投保时自行选择。

**字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 产品名称 | Text | 如"保险产品-通用""保险产品-英特尔专属"，必填 |
| 最低费率 | Number | 6 位小数，必填 |
| 最高费率 | Number | 区间费率时必填，固定费率/手工填写时为 null |
| 费率类型 | Select | 区间费率 / 固定费率 / 手工填写 |
| 是否默认 | Checkbox | 唯一默认产品，业务人员打开弹窗时自动选中 |
| 货物类型 | Text | 逗号分隔，如"全新的电子产品、电子元器件" |
| 生效日期 | Date | — |
| 失效日期 | Date | — |
| 状态 | Select | 启用 / 禁用 |
| 协议号 | Text | 专属产品关联的协议号 |
| 货值(RMB) | Number | — |
| 最低收费 | Number | — |
| 包装类型 | Select | — |
| 新旧类型 | Select | 新货 / 旧货 |
| 备注 | Textarea | — |

**费率的三种类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| 区间费率 | 配置最低和最高费率，业务人员选择后系统根据货物类型自动匹配子费率 | 0.04%–0.045% |
| 固定费率 | 单一费率值 | 0.025% |
| 手工填写 | 占位产品，不预设费率。业务人员选择后需手工填写单独报价费率及沟通备注 | 单独核定 |

**产品示例**：
- 保险产品-通用（默认） — 费率 0.04%–0.045%，货物：电子产品/元器件/精密仪器/普货
- 保险产品-英特尔专属 — 费率 0.025%，货物：英特尔CPU
- 保险产品-单独报价 — 单独核定，货物：特殊/高价值/危险品

## 5. 保险费率选择（核心交互）

### 5.1 入口

| 入口 | 位置 | 说明 |
|------|------|------|
| 投保申请表 → 更多 → 保险费率选择 | 列表页 | 弹窗选择，toast 提示已选 |
| 编辑投保单 → 保险信息区 → 保险费率选择 按钮 | 编辑页 | 弹窗选择，自动回填预计保险金额和预计保费 |

### 5.2 弹窗交互流程

```
打开弹窗
  → 展示投保单上下文（业务参考号/货物名称/货运价值/币种汇率）
  → 默认选中「通用」产品（isDefault=true）
  → 产品列表展示（货物类型标签 → 产品名 → 费率 → 预计保费）
  → 用户可选择其他产品
  → 如选「单独报价」→ 展开手工填写区（费率 + 备注）
  → 底部实时显示预计保费公式：
     货运价值 × 汇率 × 费率 = 预计保费(区间)
  → 确认选择 → 回填预计保险金额和预计保费
```

### 5.3 预计保费计算公式

```
预计保险金额 = 发票金额 × 加成比例系数

预计保费 = 预计保险金额 × 汇率 × 费率

其中：
  - 汇率：投保单币种对应的对人民币汇率（人民币=1）
  - 费率：所选产品的费率（区间费率同时计算最低和最高）
  - 区间费率 → 预计保费显示为区间：¥min – ¥max
  - 固定费率 → 预计保费显示为单值
  - 单独报价 → 手工填写费率后计算
```

### 5.4 单独报价规则

- 选中"单独报价"产品后，展开手工填写区
- 必填：单独报价费率（>0 的数字）、沟通备注（≥10 字）
- 提示文字："需提前与保险团队沟通并在备注中说明"
- 验证失败：红色边框 + 错误提示，自动滚动到错误区

## 7. 数据模型

### 7.1 核心表结构

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `insurance_applications` | 投保申请主表 | jobidref、workflow_status、6 个 JSON 段落 |
| `exchange_rates` | 保费汇率 | currency、exchange_rate、effective_date |
| `insurance_rates` | 保费费率（产品） | product_name、rate_min、rate_max、rate_type、is_default |
| `claim_reports` | 报案理赔 | report_no、report_status、claim_detail |
| `bills` | 缴费账单 | bill_no、bill_amount、bill_status |
| `invoices` | 发票 | invoice_no、invoice_amount、invoice_status |

### 7.2 JSON 段落（insurance_applications 子结构）

| 段落 | 说明 |
|------|------|
| applicantInfo | 投/被保险人信息 |
| transportInfo | 运输信息 |
| cargoInfo | 货物信息 |
| insuranceInfo | 保险信息（发票金额/币种/加成/预计保费等） |
| backfillInfo | 保单回填信息（保险公司/保单单号/实际保费） |
| correctionInfo | 批改信息 |
