# PRD -- 保险数字化管理系统 (IMS) 基础文档

> **版本**: V2.0
> **日期**: 2026-07-02
> **来源**: 生产系统 `http://43.240.126.20:51463` + Chrome DOM 捕获 + 前端路由 `/prod-api/getRouters` + 权限 `/prod-api/getInfo` + 业务弹窗与前端枚举

## 一、系统概览

| 维度 | 说明 |
|------|------|
| **系统名称** | IMS 保险数字化管理系统 |
| **生产地址** | `http://43.240.126.20:51463` |
| **技术栈** | 若敖平台 (Spring Boot + Vue) |
| **核心模块** | 基础信息 / 保单管理 / 报案理赔 |

### 导航菜单结构

```
保单管理
  └── 投保申请表 (/policyManage/insuranceapplication)

  └── 报案理赔管理 (/claimsManage/reportClaims)

  ├── 保费汇率配置 (/base/premiumexchangerateallocation)
  └── 保费费率配置 (/base/detailsoftheinsurancerateconfiguration)
```

## 二、模块与页面清单

### 模块一：保单管理 (核心模块)

| 页面 | 路由 | 核心功能 |
|------|------|----------|
| **投保申请列表** | `/policyManage/insuranceapplication` | 6 字段查询、33 列表列、导出(列表导出/平安/PICC进出口货运/PICC国内运输)、编辑/删除/发起审批/撤销审核/保险费率选择/保单信息回填/批改信息回填/缴费账单/发票/发起批改 |
| **投保详情** | 点击业务参考号进入 | 7 分组只读展示、选择附件/历史版本/审批历史/修改日志 |
| **投保编辑** | 行操作-编辑按钮 | 5 分组可编辑表单、提交/返回 |

### 模块二：报案理赔

| 页面 | 路由 | 核心功能 |
|------|------|----------|
| **报案理赔列表** | `/claimsManage/reportClaims` | 3 字段查询、10 列表列、新增/查看/理赔确认/编辑/删除/发起审批 |

### 模块三：基础信息

| 页面 | 路由 | 核心功能 |
|------|------|----------|
| **保费汇率配置** | `/base/premiumexchangerateallocation` | 保险公司+生效日期筛选、新增/编辑 |
| **保费费率配置** | `/base/detailsoftheinsurancerateconfiguration` | 保险公司+生效日期筛选、新增/批量启用/批量禁用/编辑 |

## 七、核心业务规则

1. 查询支持按投保单号(`policyNo`)、业务参考号(`jobidref`)、审批状态(`workflowStatus`)、投保时间(`applicationTime`)、保险公司保单状态(`insuranceCompanyPolicyStatus`)、保单单号(`policyNumber`)组合筛选。
2. 点击业务参考号(`jobidref`)进入投保单详情页。
3. 发票号和提运单号至少填写一个。
4. 预计保险金额 = 发票金额 x 加成比例，联动自动计算。
5. 预计保费来源于保险费率选择弹窗或单独报价输入。
6. 审批通过后，编辑/删除按钮禁用。
7. 发起审批仅支持 `workflowStatus` 为 `WaitStart`(待发起) 或 `Reject`(审批拒绝) 状态。
8. 撤销审核仅支持 `workflowStatus` 为 `Approving`(审批中) 且未承保状态。
9. 保单信息回填仅支持 `workflowStatus=Approved` 且 `submitType=1`(新增投保单)。
10. 批改信息回填仅支持 `workflowStatus=Approved` 且 `submitType=2`(批改类单据)。
11. 缴费账单、发票仅支持 `workflowStatus=Approved`(审批通过) 单据。
12. 发起批改要求 `workflowStatus=Approved` 且 `insuranceCompanyPolicyStatus=1`(已承保)。
13. 缴费账单附件和发票附件限制：文件大小不超过 5MB，允许格式 doc/docx/pdf/xls/xlsx/jpg/png/jpeg。

### 8.1 投保申请审批流转

```
                      ┌──────────────┐
                      │   草稿/Draft   │
                      └──────┬───────┘
                             │
                      ┌──────▼───────┐
                      │ 待发起/WaitStart │ ←── 可编辑/删除/保险费率选择
                      └──────┬───────┘
                             │ 发起审批(需选审批人)
                             ▼
                      ┌──────────────┐
                      │ 待审批/Approving │ ←── 撤销审核(未承保可撤)
                      └──────┬───────┘
                             │
                ┌────────────┼────────────┐
                ▼            ▼            │
         ┌──────────┐ ┌──────────┐       │
         │审批通过    │ │审批拒绝    │       │
         │Approved  │ │Reject    │       │
         └────┬─────┘ └────┬─────┘       │
              │            │             │
              │            └── 可重新编辑 ─┘
              │
     ┌────────┼────────┬────────┬────────┐
     ▼        ▼        ▼        ▼        ▼
  保单回填  缴费账单  发票管理  发起批改  ...
  (submitType=1)                (需已承保)
     │
     ▼
  回填完成(已承保)
```
